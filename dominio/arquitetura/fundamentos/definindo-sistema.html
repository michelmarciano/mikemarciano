Definindo Sistema

Definir um sistema é um processo essencial para garantir que o software seja desenvolvido de forma eficiente e eficaz. Este processo envolve várias etapas, desde a identificação dos requisitos até a definição de arquitetura e design. Abaixo estão os principais passos para definir um sistema:

Visão

No nível zero, focamos na perspectiva do negócio, que se fundamenta em dois pilares essenciais: escopo e visão. A visão é um elemento crucial que estabelece os objetivos da solução de forma empírica e lógica, definindo claramente sua razão de existência.

Um desafio comum no desenvolvimento de software ocorre quando os desenvolvedores executam tarefas sem questionar ou compreender seu impacto no negócio. Este é um problema significativo que um arquiteto de soluções não pode permitir. O arquiteto deve compreender profundamente a visão geral do projeto.

Frequentemente, a visão não está formalmente documentada. Stakeholders como proprietários ou diretores podem apresentar ideias sem detalhes específicos, mas com uma compreensão clara de como a solução beneficiará a organização. O arquiteto de soluções precisa desenvolver a habilidade de captar e interpretar essa visão, refinando-a através da experiência e do diálogo contínuo.

A visão estabelece os objetivos fundamentais que orientarão toda a solução. Por exemplo, ao criar uma startup de venda de ingressos, a visão pode incluir:

  • Interface extremamente simplificada para compra via dispositivos móveis
  • Processo de compra rápido e eficiente
  • Preços competitivos
  • Foco na retenção de clientes

Estes objetivos então guiam todas as decisões subsequentes do projeto, garantindo que cada aspecto da solução contribua para a realização da visão estabelecida.

A visão representa uma perspectiva de alto nível que abrange:

  • O que a solução realizará
  • As necessidades que atenderá
  • Os stakeholders envolvidos (funcionários, clientes, parceiros)

É fundamental que o arquiteto de soluções seja capaz de captar a essência da visão do projeto, mesmo quando ela é apresentada de forma informal ou incompleta. Esta compreensão profunda da visão é o que diferencia um arquiteto de soluções eficaz, permitindo que ele traduza conceitos abstratos em soluções concretas e alinhadas com os objetivos do negócio.

Escopo

O escopo representa a definição clara dos limites do projeto, estabelecendo o que está incluído e o que está excluído do escopo de trabalho. É neste momento que detalhamos profundamente os elementos que serão desenvolvidos.

Definição de Limites

Os limites do projeto, também conhecidos como "boundaries", são fundamentais para:

  • Estabelecer até onde a solução se estende
  • Definir o que será entregue
  • Identificar o que não faz parte da solução
  • Delimitar os contextos de atuação

Elementos do Escopo

O escopo abrange diversos aspectos essenciais:

  • Definição precisa do problema a ser resolvido
  • Requisitos funcionais e não funcionais
  • Componentes, sistemas e tecnologias a serem utilizados
  • Restrições do projeto
  • Pressupostos fundamentais

Restrições e Pressupostos

Todo projeto enfrenta diferentes tipos de restrições:

  • Restrições de negócio
  • Restrições tecnológicas
  • Restrições de tempo

Os pressupostos são premissas fundamentais para o desenvolvimento. Por exemplo, em um sistema de venda de ingressos:

  • Existência de parcerias estratégicas
  • Desenvolvimento de interface inovadora
  • Acordos com provedores de cloud computing

Transição para o Nível 1

Com a visão e o escopo bem definidos, o arquiteto de soluções pode:

  • Compreender profundamente o negócio
  • Entender a motivação por trás do projeto
  • Iniciar o planejamento da solução técnica (Nível 1)

Considerações Importantes

É importante destacar que o papel do arquiteto de soluções envolve:

  • Extenso trabalho com documentação
  • Desenvolvimento de sistemas de design
  • Gestão de contatos e cotações
  • Foco constante no entendimento do negócio

Este papel requer uma compreensão profunda tanto dos aspectos técnicos quanto dos aspectos de negócio. Embora possa parecer desafiador para profissionais com forte inclinação técnica, o entendimento do negócio e destes conceitos é fundamental para o sucesso do projeto. O arquiteto de soluções deve estar preparado para um trabalho que vai além do desenvolvimento técnico, envolvendo uma visão holística do projeto e suas implicações no negócio.
